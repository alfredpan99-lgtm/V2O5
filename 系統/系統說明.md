# 知識庫系統說明

本系統基於 Karpathy 的 LLM 知識庫架構，以 LLM 作為「編譯器」，將原始資料轉化為結構化的 Markdown 知識庫。

---

## 資料夾結構

```
/
├── raw/                  ← 第一階段：所有原始資料放這裡
├── wiki/
│   ├── 索引/             ← 自動維護的索引與摘要
│   ├── 概念/             ← 編譯後的概念文章（~100 篇）
│   └── 衍生/
│       ├── 投影片/       ← Marp 格式投影片
│       ├── 圖表/         ← matplotlib 圖表說明
│       └── 查詢回答/     ← 問答結果歸檔
└── 系統/                 ← 本說明文件與操作指南
```

---

## 四個運作階段

### 第一階段：收集（Ingest）
將原始資料放入 `raw/`：
- 用 Obsidian Web Clipper 擷取網頁文章
- 下載 arXiv 論文、GitHub repo 說明、資料集筆記
- 任何想消化的文字資料都先丟進 `raw/`

### 第二階段：編譯（Compile）
請 LLM 讀取 `raw/` 並執行以下任務：
- 更新 `wiki/索引/總索引.md`
- 為每份原始文件在 `wiki/概念/` 建立或更新概念文章
- 在文章之間加入反向連結與交叉參照

使用提示詞：見 [[編譯提示詞]]

### 第三階段：查詢與深化（Query & Enhance）
- 在 Obsidian 中瀏覽 `wiki/` 閱讀知識庫
- 向 LLM 提問複雜的研究問題，答案存入 `wiki/衍生/查詢回答/`
- 查詢結果回寫入 wiki，每次探索都累積知識

### 第四階段：校驗與維護（Lint & Maintain）
定期請 LLM 執行：
- 掃描概念不一致或資訊缺漏
- 找出概念間的潛在連結，建議新文章
- 提出下一步探索問題
- 完成後回到第二階段，wiki 持續成長

---

## 回饋迴圈

```
raw/ → 編譯 → wiki/概念/
                  ↓
              查詢回答 → 歸檔至 wiki/衍生/查詢回答/
                  ↓
              校驗與維護 → 回到編譯
```
