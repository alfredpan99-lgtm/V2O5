# 編譯提示詞

將此提示詞連同 `raw/` 內的文件一起貼給 LLM，執行第二階段編譯。

---

## 提示詞範本

```
你是一個知識庫編譯器。請閱讀以下原始文件，並執行以下任務：

## 你的任務

1. **更新總索引**
   - 在 `wiki/索引/總索引.md` 的表格中新增或更新這份文件的條目
   - 填寫：文章標題、標籤（主題分類）、一句話摘要、原始來源檔名、今天日期

2. **建立或更新概念文章**
   - 在 `wiki/概念/` 建立一篇新的 Markdown 文章
   - 文章結構：
     - `# 概念名稱`
     - `## 核心定義` — 清晰解釋這個概念是什麼
     - `## 重要細節` — 關鍵機制、數字、方法
     - `## 應用場景` — 什麼時候會用到
     - `## 相關概念` — 用 [[雙括號]] 連結到其他概念
     - `## 來源` — 原始文件連結與引用

3. **建立反向連結**
   - 找出此文件中提到的概念，如果 wiki/概念/ 裡已有對應文章，
     在那篇文章的「相關概念」區段加入指回新文章的連結

4. **提出連結建議**
   - 列出 3-5 個「此概念可能與哪些尚未建立的概念有關」

## 原始文件內容

[在此貼上 raw/ 中的文件內容]

## 輸出格式

請直接輸出可以貼入 Obsidian 的 Markdown 內容，
並在每個區塊前標示「目標檔案：wiki/概念/xxx.md」。
```

---

## 批次編譯（多份文件）

如果 `raw/` 裡有多份文件，可以一次全部提供：

```
你是一個知識庫編譯器。請依序處理以下 [N] 份原始文件，
對每份文件執行上方的編譯任務，最後輸出一份更新後的總索引表格。

文件一：[內容]
文件二：[內容]
...
```
